# Soo Solutions — Brand Mark Notes

The Soo Solutions logo system: an original, ownable mark for a security‑camera / CCTV supply &
installation company. Bright, premium, electric‑blue + deep‑navy. Built as clean, hand‑constructed SVG
(no rasterized art, no stock icons). Source of truth for the geometry: `scripts/gen-brand.mjs`.

---

## The mark — "Twin Aperture"

Two camera‑lens rings, offset on a diagonal so one sits **in front of** the other. It reads three ways at
once:

- **The "oo" of _Soo_** rendered as two lenses — a wordmark hook unique to this name (you can't lift it for
  another brand).
- **Two cameras / coverage** — residential + commercial, depth of field, "we're watching the whole
  property."
- **Forward + up** — the diagonal offset (back lower‑left → front upper‑right) reads as progress / "Safer
  Futures," not a static, symmetrical pair of eyes.

The icon **is** the abstracted "oo", so the icon and the wordmark reinforce each other.

### Concepts explored (and why this one won)

Four directions were drawn and rendered (large, at 24px favicon scale, and reversed) before choosing:

1. **Twin Aperture** ✅ — the "oo" as two offset lens rings. Most ownable, clearly a camera, tied to the
   name, doubles as the wordmark hook. **Chosen.**
2. **Aperture "S"** — a lens ring with a bold S through it. Good monogram, but the S read ambiguously (≈ a
   "2") and needed more work to be unmistakable.
3. **Forward Iris** — three sweeping blades. Read as a loading spinner / fan, not a camera. Rejected.
4. **Safeguard Lens** — a lens nested in a soft keystone cradle. Drifted toward the generic "CCTV eye /
   target" cliché the brief forbids. Rejected.

Deliberately avoided clichés: plain shield + checkmark, a literal little camera, a lone location pin,
Wi‑Fi/signal waves, a single circle‑lens, a staring CCTV eye.

### Construction

Built on a **100‑unit grid** with a single consistent stroke weight:

| Element | Value |
| --- | --- |
| Lens ring radius | `21` |
| Stroke weight (both rings) | `8` (consistent) |
| Pupil radius | `5` |
| Back lens centre | `(37, 63)` |
| Front lens centre | `(63, 37)` — a balanced diagonal offset (`Δ 26, 26`) |
| Depth | front lens masks the back ring where they overlap (clean front‑over‑back) |

The mark sits in a `~12 → 88` box inside the 100‑grid, giving built‑in optical breathing room. The favicon
keeps the **same** geometry (no separate simplified mark) so it stays consistent and crisp from 16px up.

## Colour tokens

The logo uses the site design tokens, so it adapts to light and dark automatically (front lens = `primary`,
back lens = `foreground`).

| Role | Token | OKLCH | Hex |
| --- | --- | --- | --- |
| Front lens — electric blue | `--primary` | `oklch(0.55 0.218 256)` | `#2563eb` |
| Back lens / pupils — deep navy | `--foreground` | `oklch(0.255 0.045 264)` | `#1c2540` |
| Surface — warm near‑white | `--background` | `oklch(0.985 0.006 95)` | `#fafaf7` |
| Reversed accent (on navy) | — | `oklch(0.74 0.13 256)` | `#7db0ff` |
| App‑icon container | `--primary` | `oklch(0.55 0.218 256)` | `#2563eb` |

**Contrast:** the navy back lens on warm near‑white and the white mark on the blue app‑icon both clear WCAG
AA for non‑text UI. On dark/navy surfaces use the **reversed** variant (white front + light‑blue back).

| `lockup-stacked.svg` | Mark above "Soo Solutions" — square‑ish placements |
| `wordmark.svg` | "Soo Solutions" alone |

The **wordmark** is set in **Space Grotesk** (the site display face; pairs with DM Sans for body), bold,
`-0.02em` tracking. On the website the wordmark is live, accessible text (`Logo.tsx`, `font-display`); the
standalone SVGs reference Space Grotesk with a geometric system‑sans fallback.

In React, use `Logo.tsx`:

- `<Logo />` — horizontal lockup (header/footer).
- `<Logo variant="icon" />` — mark only, links home.
- `<Logo variant="stacked" />` — stacked.
- `<LogoMark />` — bare SVG mark (decorative use, e.g. 404).

Every form carries `<title>` / `aria-label` **"Soo Solutions"**.

Favicon / app icons (generated by `scripts/gen-brand.mjs`): `src/app/icon.svg`, `src/app/apple-icon.png`,
`src/app/favicon.ico`, `public/icon-192.png`, `public/icon-512.png`, `public/icon-maskable-512.png`, and
the dynamic OG image (`src/lib/og.tsx`).

## Clear space & minimum size

- **Clear space:** keep free space on all sides equal to **one lens radius** (≈ 25% of the mark's width).
  Nothing — text, edges, other logos — intrudes into that zone.
- **Minimum size:** mark **16px** (favicon / app‑icon square); **20px** for the freestanding 2‑colour mark;
  horizontal lockup **≥ 24px tall** (mark) so the wordmark stays legible.
- **Don't:** recolour outside the tokens, add effects/shadows/gradients to the mark, stretch it, rotate it,
  outline it, or place the 2‑colour mark on a busy/low‑contrast photo (use the reversed variant on a solid
  navy panel instead).

## Originality & trademark — be honest

A quick **web + image sanity check** was run on the chosen concept ("two overlapping camera‑lens rings as
the _oo_ of a security‑camera wordmark"). It surfaced **dual‑lens camera _products_** (Reolink, Ring, Lorex)
and generic "oo lens" design _templates_, but **no existing brand mark** that clashes — and nothing
resembling the partner logos (Lorex, Hikvision, HiLook). The mark was also designed to **not imitate** those
partners.

**This is a sanity check, NOT legal clearance.** Designing for distinctiveness reduces, but does not
eliminate, conflict risk. Before claiming ™ / ® or registering this mark, the client should:

1. Run a formal search of the **CIPO (Canadian Intellectual Property Office) trademarks database** for
   Canada,
2. Search the **USPTO** (and any other relevant jurisdictions) for marks in the same classes,
3. Run a broad **web / image search**, and
4. **Consult a trademark professional** before relying on the mark commercially.
